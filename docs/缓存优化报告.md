# 缓存优化报告 - v1.4

**执行日期**: 2026-03-26  
**执行者**: 犀甲  
**版本**: v1.3 → v1.4

---

## 优化目标

解决高频查询的性能问题：
- 年假余额查询（每次都要计算）
- 员工列表查询（变化不频繁）
- 减少飞书 API 调用次数

---

## 实现方案

### 1. 缓存模块 (cache.py)

```python
# 核心组件
- MemoryCache: 带 TTL 的内存缓存（线程安全）
- @cached: 装饰器自动缓存函数结果
- LeaveCache: 年假查询专用缓存封装
```

**TTL 配置**:
| 数据类型 | 缓存时间 | 说明 |
|----------|----------|------|
| 年假余额 | 30分钟 | 计算成本高，变化少 |
| 员工列表 | 5分钟 | 变化较少 |
| 请假明细 | 10分钟 | 可能有实时变化 |
| 导出数据 | 10分钟 | 批量计算成本高 |

---

### 2. API 集成

**年假余额查询**:
```python
# 1. 尝试从缓存获取
cached_result = cache.get(cache_key)
if cached_result:
    return cached_result  # 缓存命中

# 2. 计算年假余额
result = calculate_leave_balance(...)

# 3. 存入缓存
cache.set(cache_key, result, ttl=1800)
```

**调整记录创建**:
```python
# 创建调整记录后，使相关缓存失效
LeaveCache.invalidate_balance_by_name(employee_name, year)
```

## 技术亮点

### 1. 线程安全
```python
self._lock = threading.RLock()
# 所有缓存操作加锁，支持并发
```

### 2. 自动过期
```python
# TTL 自动管理，无需手动清理
expire_time = time.time() + ttl
cache.set(key, value, ttl=1800)
```

### 3. 缓存失效策略
```python
# 数据更新时自动失效相关缓存
@post('/api/admin/adjustments')
def create_adjustment(...):
    db.create_adjustment(...)
    LeaveCache.invalidate_balance(employee_id, year)  # 自动失效
```
