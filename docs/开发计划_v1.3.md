## 2. 已确认业务规则

| 规则 | 说明 |
|------|------|
| 年假发放 | 每年元旦（1月1日）一次性入账，支持透支，余额可为负数 |
| 司龄递增 | 司龄增加当天，年假额度递增1天 |
| 年终清算 | 每年12月31日清算，余额最多结转3天到次年3月31日 |

| 额度封顶 | 普通员工12天封顶，老员工15天封顶 |
| 最小请假单位 | 0.5天，属公司制度 |
| 不设提醒 | 年假按需休，尊重员工意愿 |

#### P0: 飞书免登自动识别用户
**问题**: 当前通过下拉框选择员工，任何人能查看所有人年假信息，存在隐私问题

**方案**:
- 前端：引入飞书 JSSDK，调用 `tt.requestAuthCode()` 获取免登授权码，移除员工下拉框
- 后端：新增 `/api/auth/login` 接口，用 code 换取 user_access_token → 获取 open_id → 匹配员工
- 员工表：多维表格增加「飞书Open ID」字段
- HR后台：保留员工选择器，HR角色验证后可查看/调整他人数据

**验收标准**:
- 员工打开应用自动显示自己的年假信息
- 普通员工只能看到自己的数据
- HR角色可切换查看其他员工

**预估工时**: 3-4h

**详细实现**:

```python
# backend/export.py
import csv
import io
from fastapi import Response
from openpyxl import Workbook

@router.get("/api/admin/export")
async def export_leave_data(
    year: int = Query(default=None),
    format: str = Query(default="csv", regex="^(csv|xlsx)$"),
    current_user: User = Depends(require_hr)
):
    """
    导出全员年假数据
    """
    # 1. 获取所有员工
    employees = await get_all_employees()
    
    # 2. 计算每人年假数据
    data = []
    for emp in employees:
        leave_info = calculate_annual_leave(emp, year)
        data.append({
            "姓名": emp.name,
            "工号": emp.employee_id,
            "入职日期": emp.entry_date,
            "司龄(月)": emp.service_months,
            "当年额度": leave_info.current_year,
            "上年结转": leave_info.carryover,
            "已用天数": leave_info.used,
            "余额": leave_info.remaining,
            "调整记录": leave_info.adjustments_summary
        })
    
    # 3. 生成文件
    if format == "csv":
        output = io.StringIO()
        writer = csv.DictWriter(output, fieldnames=data[0].keys())
        writer.writeheader()
        writer.writerows(data)
        content = output.getvalue()
        media_type = "text/csv"
        filename = f"年假数据_{year}.csv"
    else:
        wb = Workbook()
        ws = wb.active
        ws.append(data[0].keys())
        for row in data:
            ws.append(row.values())
        output = io.BytesIO()
        wb.save(output)
        content = output.getvalue()
        media_type = "application/vnd.openxmlformats-officedocument.spreadsheetml.sheet"
        filename = f"年假数据_{year}.xlsx"
    
    return Response(
        content=content,
        media_type=media_type,
        headers={"Content-Disposition": f"attachment; filename={filename}"}
    )
```

```python
async def preview_year_end_settlement(

        "details": preview
    }

    return {"success": True, "settlement_id": settlement.id}
```

#### P1-5: 司龄递增自动化 🆕
**问题**: 规则说"司龄增加当天年假递增1天"，但系统只读取多维表格的固定值

**确认结论**: ✅ 采用方案A - 系统根据入职日期自动计算司龄

**方案**:
- 根据员工「入职时间」字段，系统自动计算司龄（月数）
- 不再依赖多维表格的「司龄(月)」字段
- 实现 `calculate_service_months(entry_date, current_date)` 函数

**验收标准**:
- 员工司龄满年时，年假额度自动+1天
- 无需HR手动修改多维表格

**预估工时**: 2h

**详细实现**:

```python
# backend/leave_calculator.py - 修改司龄计算

class LeaveCalculator:
    def __init__(self):
        self.rules = LEAVE_RULES
    
    def calculate_service_months(self, entry_date: date, current_date: date) -> int:
        """
        根据入职日期计算司龄（月）
        """
        if not entry_date:
            return 0
        
        months = (current_date.year - entry_date.year) * 12
        months += current_date.month - entry_date.month
        
        # 如果当前日期还没到入职日，减去1个月
        if current_date.day < entry_date.day:
            months -= 1
        
        return max(0, months)
    
    def calculate_welfare_leave(self, entry_date: date, current_date: date) -> int:
        """
        计算公司福利年假（基于入职日期自动计算司龄）
        """
        service_months = self.calculate_service_months(entry_date, current_date)
        welfare = self.rules["welfare_leave"]
        
        # 新员工（<12个月）
        if service_months < welfare["new_employee_months"]:
            return welfare["new_employee_days"]
        
        # 司龄年数
        service_years = service_months // 12
        
        # 中高级员工（12-180个月）
```

## 11. 风险点与注意事项

| 风险 | 影响 | 应对措施 |
|------|------|---------|
| 员工表缺少入职日期 | 司龄计算失败 | 添加兼容性处理，回退到读取表格字段 |

| 导出数据量大 | 性能问题 | 分页导出，或异步导出+通知下载 |
| 员工 Open ID 匹配失败 | 无法登录 | 提供手动绑定界面 |
