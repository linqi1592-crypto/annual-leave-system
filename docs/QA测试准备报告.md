## 一、项目理解总结

### 1.1 项目概述
年假查询系统是一个部署在飞书工作台上的轻量级应用，让员工能够：
- 实时查询个人年假余额（法定+福利+结转）
- 查看本年度请假明细时间轴
- 了解年假计算规则明细

同时提供 HR 后台功能：
- 手工调整上年剩余年假（处理误操作）
- 调整记录留痕，支持审计追溯

### 1.2 技术架构
```
飞书工作台 → FastAPI 服务 → 飞书多维表格 + SQLite
```

**关键组件：**
- **后端**: Python + FastAPI
- **数据源**: 飞书多维表格（员工信息、请假记录）
- **本地存储**: SQLite（调整记录）
- **部署**: 服务器 + 飞书应用集成

### 1.3 核心功能模块

| 模块 | 功能 | 用户 |
|------|------|------|
| 年假余额查询 | 展示法定/福利/结转/总额 | 员工 |
| 请假明细 | 时间轴展示请假记录 | 员工 |
| 规则说明 | 展示计算逻辑明细 | 员工 |
| 调整管理 | 增删改查调整记录 | HR |

### 1.4 业务规则复杂度
系统涉及 **6 大类计算规则**：
1. 法定年假（按社保工龄）
2. 福利年假（按司龄）
3. 封顶规则（一般12天/老员工15天）
4. 结转规则（上年最多转3天，3月31日到期）
5. 入职折算（按入职后剩余天数比例）
6. 离职折算（按在职天数比例，超休扣回）

## 三、测试策略概述

### 3.1 测试层次

```
┌─────────────────────────────────────┐
│  集成测试（API + 数据源 + 计算引擎）  │  ← 重点
├─────────────────────────────────────┤
│  单元测试（计算函数边界值）           │
├─────────────────────────────────────┤
│  冒烟测试（核心接口可用性）           │
└─────────────────────────────────────┘
```

### 3.2 测试方法

#### 1. 静态测试
- [ ] 代码 Review（已完成）
- [ ] 配置文件检查（.env 示例完整性）
- [ ] 接口文档一致性检查

#### 2. 动态测试
- [ ] **冒烟测试**: 验证 7 个接口基本可用
- [ ] **功能测试**: 验证业务逻辑正确性
- [ ] **边界测试**: 工龄/司龄临界值、年假天数边界
- [ ] **异常测试**: 无效参数、鉴权失败、数据源异常
- [ ] **集成测试**: 完整业务流程（查余额 → 请年假 → 再查余额）

#### 3. 数据驱动测试
准备 **测试数据集**：
- 新员工（司龄<12月，工龄<12月）
- 普通员工（司龄1-15年，工龄1-15年）
- 资深员工（司龄>15年，工龄15-20年）
- 老员工（工龄>20年）
- 年中入职员工
- 年中离职员工
- 有结转记录员工
- 有调整记录员工

### 3.3 测试环境

| 环境 | 用途 | 数据源 |
|------|------|--------|
| 本地开发环境 | 开发调试 | 测试多维表格 |
