## 二、关键测试场景

### TC-POOL-004: 并发访问测试 ⭐⭐⭐⭐⭐
**目的**: 验证连接池解决 SQLite database locked 问题

**步骤**:
1. 初始化 3 个连接的连接池
2. 启动 5 个线程同时操作数据库
3. 每个线程创建表、插入数据、提交

**预期**: 所有线程成功，无 database locked 错误

**结果**: ✅ 通过

---

### TC-DB-003: 并发写入测试 ⭐⭐⭐⭐⭐
**目的**: 模拟年终清算时多 HR 并发调整场景

**步骤**:
1. 创建 adjustments 表
2. 5 个线程同时写入，每个写入 3 条
3. 线程间有 10ms 延迟模拟真实场景

**预期**: 
- 无错误
- 最终数据 15 条（5×3）

**结果**: ✅ 通过

---

### TC-INT-002: 并发 API 调用模拟 ⭐⭐⭐⭐⭐
**目的**: 验证缓存+连接池在高并发下的稳定性

**步骤**:
1. 10 个线程同时模拟 API 调用
2. 每个线程检查缓存 → 查询/写入数据库 → 更新缓存

**预期**: 
- 所有线程成功
- 数据完整性正确

**结果**: ✅ 通过
