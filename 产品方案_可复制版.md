# 年假查询系统 - 产品方案

## 一、项目背景

**痛点：**
- 员工不知道今年还剩几天年假
- HR 每月被多次咨询年假余额
- 年假计算规则复杂（法定+福利+结转+折算），员工难以理解
- 历史数据误操作需要修正，缺乏留痕机制

**目标：**
提供飞书工作台上的轻量级年假查询应用，让员工自助查询余额和明细。

---

## 二、产品方案

### 核心功能

#### 员工端（C 端）
| 功能 | 说明 |
|------|------|
| **余额卡片** | 展示法定/福利/结转/总额，剩余天数一目了然 |
| **请假明细** | 时间轴展示本年度所有请假记录 |
| **规则说明** | 解释年假计算逻辑，提升透明度 |

#### HR 后台（B 端）
| 功能 | 说明 |
|------|------|
| **手工调整** | 修正上年剩余年假（处理误操作）|
| **调整记录** | 记录调整人、调整量、原因，支持审计 |
| **数据导出** | 导出调整记录用于合规审计 |

---

## 三、技术架构

```
┌─────────────────┐
│   飞书工作台     │ ← 员工入口
└────────┬────────┘
         │
┌────────▼────────┐
│   FastAPI 服务   │ ← 部署在公司服务器
│  - 年假计算引擎  │
│  - 6 个 API 接口 │
└────────┬────────┘
         │
    ┌────┴────┐
    ▼         ▼
┌────────┐ ┌──────────┐
│飞书多维表格│ │ SQLite   │
│(员工+请假)│ │(调整记录) │
└────────┘ └──────────┘
```

**架构优势：**
- 复用飞书现有数据源，无需重复录入
- 轻量级部署，运维成本低
- 敏感调整记录存本地，保障数据安全

## 四、数据源

### 1. 员工信息表（飞书多维表格）
| 字段 | 用途 |
|------|------|
| 发起人 | 员工姓名 |
| 入职时间 | 计算司龄、入职折算 |
| 离职时间 | 判断是否在职、离职折算 |
| 司龄(月) | 计算福利年假 |
| 工龄(月) | 计算法定年假 |

### 2. 请假记录表（飞书多维表格）
| 字段 | 用途 |
|------|------|
| 申请编号 | 唯一标识 |
| 申请状态 | 已通过/已撤回（参与计算）|
| 发起人 | 关联员工 |
| 请假类型 | 年假/事假等 |
| 时长 | 请假天数 |

### 3. 调整记录表（本地 SQLite）
| 字段 | 说明 |
|------|------|
| employee_name | 员工姓名 |
| year | 调整年度 |
| adjust_amount | 调整金额（+/-）|
| reason | 调整原因 |
| created_by | 操作人 |
| created_at | 操作时间 |
| is_active | 软删除标记 |

## 五、年假计算规则

### 1. 法定年假（按社保工龄）
| 工龄 | 天数 |
|------|------|
| < 1 年 | 0 天 |
| 1-15 年 | 5 天 |
| 15-20 年 | 10 天 |
| > 20 年 | 15 天 |

### 2. 福利年假（按司龄）
| 司龄 | 天数 |
|------|------|
| < 1 年 | 6 天（新人福利）|
| 1-15 年 | 1 + 司龄年数 |
| ≥ 15 年 | 司龄年数 |

### 3. 封顶规则
- 一般员工：最多 12 天
- 老员工（工龄>20 年）：最多 15 天

### 4. 结转规则
- 上年剩余最多结转 3 天
- 结转年假次年 3 月 31 日到期

### 5. 折算规则
- **入职**：按当年剩余天数比例折算
- **离职**：按当年在职天数比例折算，已休超出部分扣回

## 六、API 接口

| 方法 | 接口 | 功能 |
|------|------|------|
| GET | /api/leave/balance | 查询年假余额 |
| GET | /api/leave/details | 查询请假明细 |
| GET | /api/leave/rules | 查询计算规则 |
| GET | /api/adjustments | HR 查询调整记录 |
| POST | /api/adjustments | HR 新增调整 |
| DELETE | /api/adjustments/{id} | HR 删除调整 |

## 九、部署清单

### 环境要求
- Python 3.8+
- 支持外网访问的服务器

### 部署步骤
```bash
# 1. 拉取代码
git clone https://github.com/linqi1592-crypto/annual-leave-system.git
cd annual-leave-system/backend

# 2. 配置环境变量
cp .env.example .env
# 编辑 .env，填入 FEISHU_APP_SECRET

# 3. 安装依赖
pip install -r requirements.txt

# 4. 启动服务
python main.py
```

### 飞书配置
- 网页应用主页：`http://服务器地址:8000`
- 配置可见范围：全员或指定部门

## 十、验证计划

**验证方式：**
- 选取 3-5 名年假情况复杂的员工（含新入职、有结转、有调整等情况）
- 对比系统计算结果与人工计算结果
- 确认无误后全员上线

---

## 十一、预期收益

**员工体验：**
- 自助查询，无需等待 HR 回复
- 计算逻辑透明，减少误解

**HR 效率：**
- 减少重复咨询
- 调整记录留痕，审计合规

**成本：**
- 系统成本：低（复用飞书基础设施）
- 运维成本：低（单服务部署）
